Tarea: Principios de Ingeniería de Software

Este repositorio tiene tres ejemplos básicos en C# que muestran tres principios importantes de la ingeniería de software: DRY, KISS y YAGNI.

Ejemplo DRY

Este ejemplo muestra el principio DRY, que básicamente dice que no hay que repetir la misma lógica en varios lugares del código, en vez de escribir el cálculo de precio por cantidad cada vez que lo necesitamos, lo metemos en un método y lo llamamos cuantas veces queramos, así si algo cambia, solo lo cambiamos en un lugar y listo, no hay que buscar por todo el código dónde lo escribimos de nuevo.

Ejemplo KISS

Este ejemplo muestra el principio KISS, que nos dice que hay que mantener las cosas simples, el método solo suma dos números y ya, nada más, no tiene condiciones raras ni lógica de más, cuando el código es así de directo, cualquiera lo puede leer y entender rápido, y si hay un error también es más fácil encontrarlo.

Ejemplo YAGNI

Este ejemplo muestra el principio YAGNI, que nos recuerda que no hay que agregar cosas al código que todavía no se necesitan. El método saluda a una persona por su nombre y nada más, no tiene opciones de idioma, ni formato especial, ni nada extra porque eso no se pidió. Agregar cosas de más solo complica el código sin ningún beneficio real.
